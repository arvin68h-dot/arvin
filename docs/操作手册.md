## 1. 概述

### 1.1 什么是 CodeEngine

CodeEngine 是一个全场景 AI 编码引擎，能够：

- **多语言支持**：C++、Python、JavaScript/TypeScript、Go、Rust、C#、PowerShell、CATIA
- **代码操作**：读写文件、编辑代码、搜索代码库
- **编译执行**：自动编译、运行、调试代码
- **项目管理**：Git 版本控制、CMake 构建管理
- **工程开发**：CATIA 二次开发、VBScript 宏编写
- **本地运行**：支持本地 LLM 离线推理

### 1.2 系统架构

```
用户 (CLI/CodeWorker)
    │
    ▼
┌─────────────────────────┐
│   CodeEngine CLI        │  ← 命令解析与路由
│   ┌─────────────────┐   │
│   │   ACP Host      │───┼──→ CodeWorker 集成 (JSON-RPC)
│   └─────────────────┘   │
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│   工具注册表            │  ← 统一管理 19 个工具
│   read_file / write     │
│   edit_file / shell     │
│   search / git          │
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│   语言引擎              │  ← 编译/格式化/Lint
│   C++ / Python / JS     │
│   TypeScript / Go / ... │
└─────────┬───────────────┘
          │
          ▼
┌─────────────────────────┐
│   Provider (LLM)        │  ← AI 模型通信
│   Ollama / OpenAI       │
└─────────────────────────┘
```

## 2. 快速开始

### 2.1 三步上手

```bash
# 第一步：安装依赖
pnpm install

# 第二步：编译项目
npx tsc --build

# 第三步：运行
npx tsx cli/src/main.ts --version
```

### 2.2 配置 LLM 后端

编辑 `~/.codeengine/config.toml`：

```toml
[providers.ollama]
id = "ollama"
type = "ollama"
name = "Ollama"
model = "qwen3.6-35b"        # 替换为你的模型名
baseURL = "http://10.0.0.11:1234"
maxTokens = 8192
temperature = 0.7
```

### 2.3 启动对话

```bash
npx tsx cli/src/main.ts run
```

## 3. 安装与部署

### 3.1 环境要求

| 项目 | 最低版本 | 推荐版本 |
|------|----------|----------|
| Node.js | v22.0 | v22.22+ |
| pnpm | v8.0 | v9+ |
| 内存 | 4 GB | 8 GB+ |
| 磁盘 | 2 GB | 5 GB+ |

### 3.2 安装 Node.js

```bash
# 方式一：通过 nvm (推荐)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash
nvm install 22
nvm use 22

# 方式二：从官网下载
# https://nodejs.org/en/download/
```

### 3.3 安装 pnpm

```bash
npm install -g pnpm
pnpm --version   # 验证安装
```

### 3.4 安装 CodeEngine

```bash
# 从 Git 克隆
git clone <repository-url>
cd codeengine

# 安装依赖
pnpm install --frozen-lockfile

# 编译
npx tsc --build
```

### 3.5 验证安装

```bash
# 检查版本
npx tsx cli/src/main.ts --version

# 检查引擎状态
npx tsx cli/src/main.ts engine list

# 检查工具状态
npx tsx cli/src/main.ts tool list
```

### 3.6 依赖工具安装

CodeEngine 依赖外部编译器和工具：

```bash
# C++ 编译器 (macOS)
brew install gcc          # 安装 g++

# C++ 编译器 (Linux)
apt install g++           # Debian/Ubuntu
yum install gcc-c++       # CentOS/RHEL

# Go
brew install go           # macOS
apt install golang-go     # Linux

# Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Python
brew install python3      # macOS
apt install python3       # Linux

# Git
brew install git          # macOS
apt install git           # Linux
```

## 4. 配置管理

### 4.1 配置文件位置

| 类型 | 路径 | 说明 |
|------|------|------|
| 用户配置 | `~/.codeengine/config.toml` | 全局配置 |
| 项目配置 | `.codeengine/config.toml` | 项目级覆盖 |
| 数据存储 | `~/.codeengine/data/codeengine.db` | SQLite 数据库 |
| 日志目录 | `~/.codeengine/logs/` | 日志文件 |
| 技能目录 | `~/.codeengine/skills/` | 自定义技能 |

### 4.2 配置文件结构

```toml
# ── 通用设置 ──
[general]
log_level = "info"              # 日志级别: trace|debug|info|warn|error|silent
log_to_file = true              # 是否写日志文件
log_dir = "~/.codeengine/logs"  # 日志目录
max_concurrent_requests = 1     # 最大并发请求数
streaming_enabled = true        # 是否启用流式输出

# ── 会话设置 ──
[session]
auto_save = true                # 自动保存会话
save_interval = 30000           # 保存间隔 (毫秒)
max_messages = 10000            # 会话最大消息数
timeout = 3600000               # 会话超时 (毫秒)

# ── LLM 提供商 ──
[providers.default]
value = "ollama"                # 默认提供商

[providers.ollama]
id = "ollama"
type = "ollama"
name = "Ollama"
model = "qwen3.6-35b"
baseURL = "http://10.0.0.11:1234"
maxTokens = 8192
temperature = 0.7
topP = 0.9
timeout = 120000

# ── 存储设置 ──
[storage]
path = "~/.codeengine/data/codeengine.db"
vacuumInterval = 86400000       # 数据库清理间隔 (毫秒)

# ── 快照设置 ──
[checkpoint]
auto_before_shell = true        # Shell 操作前自动快照
max_snapshots = 20              # 最大快照数
backup_dir = ".codeengine.backup"

# ── 压缩设置 ──
[compaction]
strategy = "hybrid"             # 压缩策略
maxInputTokens = 128000         # 最大输入 token 数
targetTokens = 64000            # 目标 token 数
minTokens = 16000               # 最小 token 数

# ── 权限设置 ──
[permission]
default = "ask"                 # 默认策略: allow|deny|ask
always_allow = ["read_file", "glob", "list_dir"]
always_deny = ["sudo"]

# ── 引擎设置 ──
[engine]
[engine.compilers]
gpp = "g++"
gcc = "gcc"
python = "python3"
node = "node"

# ── 技能设置 ──
[skills]
dir = "~/.codeengine/skills"

# ── 功能开关 ──
[features]
codebase_search = true
lsp_integration = true
checkpoint_system = true
skill_system = true
task_planning = true
mcp_support = true
```

### 4.3 修改配置

```bash
# 查看配置
npx tsx cli/src/main.ts config get general.log_level

# 设置配置项
npx tsx cli/src/main.ts config set general.log_level debug

# 重置为默认配置
npx tsx cli/src/main.ts config reset
```

### 4.4 环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `CODEENGINE_LOG_LEVEL` | 日志级别 | `info` |
| `CODEENGINE_WORKSPACE` | 工作目录 | 当前目录 |
| `CODEENGINE_CONFIG` | 配置文件路径 | `~/.codeengine/config.toml` |

### 6.2 文件工具

#### read_file — 读取文件

```
工具: read_file
类别: 文件操作
描述: 读取文件内容，支持行范围过滤
```

**参数:**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| path | string | 是 | 文件路径 |
| offset | number | 否 | 起始行号 (默认: 1) |
| limit | number | 否 | 返回行数 (默认: 100) |

**示例:**
```bash
# 读取整个文件
npx tsx cli/src/main.ts run  # 然后输入: "读取 hello.py 的内容"

# 读取前 50 行
代码: read_file(path="hello.py", offset=1, limit=50)
```

#### write_file — 写入文件

```
工具: write_file
类别: 文件操作
描述: 写入文件内容，自动创建父目录
```

**参数:**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| path | string | 是 | 文件路径 |
| content | string | 是 | 文件内容 |

**安全特性:**
- 自动创建父目录
- 路径安全检查（防止写入系统目录）
- 写操作日志记录

**示例:**
```
输入: "创建一个 Python 文件 hello.py，内容为 print('Hello World')"
```

#### edit_file — 编辑文件

```
工具: edit_file
类别: 文件操作
描述: 支持三种编辑模式
```

**编辑模式:**

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| exactReplace | 精确替换指定文本 | 修改明确代码片段 |
| findAndReplace | 查找并替换 | 批量替换常量/变量名 |
| insertAtLine | 指定行插入 | 在特定位置添加代码 |

**参数:**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| path | string | 是 | 文件路径 |
| mode | string | 是 | 编辑模式 |
| oldString | string | exactReplace 需要 | 要替换的原文 |
| newString | string | exactReplace 需要 | 新文本 |
| findPattern | string | findAndReplace 需要 | 查找模式 |
| line | number | insertAtLine 需要 | 插入行号 |
| content | string | insertAtLine 需要 | 插入内容 |

#### delete_file — 删除文件

```
工具: delete_file
类别: 文件操作
描述: 删除文件，包含路径安全检查
```

**安全限制:**
- 禁止删除系统目录中的文件
- 禁止递归删除整个目录
- 删除前自动确认路径安全性

#### list_dir — 列出目录

```
工具: list_dir
类别: 文件操作
描述: 列出目录内容，支持递归
```

**参数:**

| 参数 | 类型 | 必填 | 说明 |

## 7. 语言引擎

### 7.1 引擎架构

每个语言引擎继承基类 `LanguageEngine`，实现以下接口：

```typescript
interface LanguageEngine {
  /** 构建/编译 */
  build(sourcePath: string, options?: BuildOptions): Promise<BuildResult>;

  /** 代码格式化 */
  format(code: string, filePath?: string): Promise<string>;

  /** 静态检查 */
  lint(code: string, filePath?: string): Promise<LintResult>;

  /** 代码分析 */
  analyze(code: string, filePath?: string): Promise<AnalysisResult>;
}
```

### 7.2 引擎列表

| 引擎 | 文件位置 | 用途 | 依赖 |
|------|----------|------|------|
| base | `base/index.ts` | 抽象基类 | 无 |
| cpp | `cpp/index.ts` | C++ 编译运行 | g++ |
| python | `python/index.ts` | Python 脚本运行 | python3 |
| js | `js/index.ts` | JavaScript 运行 | node |
| ts | `ts/index.ts` | TypeScript 编译 | tsc |
| go | `go/index.ts` | Go 编译运行 | go |
| rust | `rust/index.ts` | Rust 编译运行 | cargo/rustc |
| cs | `cs/index.ts` | C# 编译运行 | dotnet |
| ps | `ps/index.ts` | PowerShell 执行 | pwsh |
| cmake | `cmake/index.ts` | CMake 构建 | cmake |
| catia | `catia/index.ts` | CATIA 脚本 | VBScript 解析 |

### 7.3 各引擎详解

#### C++ 引擎

```typescript
// 支持特性
- g++/clang++ 编译
- 自动检测 Makefile/CMakeLists.txt
- 错误输出解析
- 可执行文件运行
```

**使用示例:**
```
输入: "创建一个 C++ hello world 程序"

代码引擎自动:
1. 创建 hello.cpp 文件
2. 执行 g++ hello.cpp -o hello
3. 运行 ./hello
4. 返回输出: "Hello, World!"
```

#### Python 引擎

```typescript
// 支持特性
- python3 执行
- 依赖检测 (requirements.txt)
- 异常堆栈解析
- 虚拟环境支持
```

**使用示例:**

## 8. 会话管理

### 8.1 会话概念

会话是 AI 对话的上下文容器，包含：

- **消息历史**: 用户与 AI 的对话记录
- **系统提示**: AI 的角色和行为规范
- **上下文压缩**: 消息过多时自动压缩历史
- **持久化**: 自动保存到 SQLite

### 8.2 会话操作

```bash
# 列出所有会话
npx tsx cli/src/main.ts session list

# 查看会话详情
npx tsx cli/src/main.ts session show <sessionId>

# 创建新会话
npx tsx cli/src/main.ts session new

# 清空当前会话
npx tsx cli/src/main.ts session clear
```

### 8.3 上下文压缩

当会话消息超过 `maxMessages` 配置或 token 数超过阈值时，系统自动触发上下文压缩：

```
触发条件:
  - token 数 > maxInputTokens (默认 128000)
  - token 数 > minTokens (默认 16000)

压缩策略:
  - merge: 合并相似消息
  - summarize: 生成摘要
  - hybrid: 混合策略 (推荐)

压缩结果:
  - 保留最近 N 条消息 (最近优先)
  - 对旧消息生成摘要
  - 保留关键代码和错误信息
```

## 12. CodeWorker 集成

### 12.1 ACP 协议

CodeEngine 支持通过 ACP (Agent Communication Protocol) 与外部系统通信。

**协议规范:**
- 通信方式: 标准输入/输出 (stdin/stdout)
- 格式: JSON-RPC 2.0
- 编码: UTF-8
- 分隔符: 每行一个 JSON 对象 (NDJSON)

### 12.2 启动 CodeEngine 插件模式

```bash
# 方式一：使用集成脚本
./scripts/codeworker-integration.sh

# 方式二：直接启动
npx tsx cli/src/acp-host.ts

# 方式三：调试模式
CODEENGINE_LOG_LEVEL=debug npx tsx cli/src/acp-host.ts
```

### 12.3 支持的 ACP 方法

| 方法 | 说明 | 参数 | 响应 |
|------|------|------|------|
| `task/execute` | 执行编码任务 | task, workspace, messages | taskId, status, output |
| `task/cancel` | 取消任务 | taskId | status: cancelled |
| `tool/list` | 列出工具 | 无 | 工具列表 |
| `engine/list` | 列出引擎 | 无 | 引擎列表 |
| `session/new` | 创建会话 | 无 | sessionId |
| `session/history` | 获取历史 | sessionId | 消息列表 |

### 12.4 请求示例

```json
// 请求
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "task/execute",
  "params": {
    "task": "创建一个 Python 加法函数",
    "workspace": "/path/to/project"
  }
}

// 响应
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "taskId": "task-abc123",
    "status": "completed",
    "steps": [
      "分析任务: 创建一个 Python 加法函数",
      "创建 add.py",
      "执行代码验证"
    ],
    "output": "任务完成: add.py 已创建"
  }
}
```

## 14. 故障排查

### 14.1 常见错误

**错误 1: 编译失败**

```
错误信息: npx tsc --build 报错
解决方案:
1. 检查 Node.js 版本: node --version (需要 v22+)
2. 清理编译缓存: npx tsc --build --clean
3. 重新安装依赖: pnpm install
4. 再次编译: npx tsc --build
```

**错误 2: 无法连接到 LLM**

```
错误信息: connect ECONNREFUSED
解决方案:
1. 确认 Ollama 服务运行中: ollama list
2. 检查 baseURL 配置是否正确
3. 测试连接: curl http://localhost:1234/api/tags
4. 检查防火墙设置
```

**错误 3: 编译工具未找到**

```
错误信息: g++: command not found
解决方案:
macOS: brew install gcc
Linux: apt install g++ / yum install gcc-c++
Windows: 安装 MinGW 或 Visual Studio Build Tools
```

**错误 4: 权限被拒绝**

```
错误信息: EACCES: permission denied
解决方案:
1. 检查文件权限: ls -la <file>
2. 修复权限: chmod 644 <file>
3. 检查目录权限: ls -la <dir>
4. 检查配置文件的 permission 设置
```

### 14.2 日志调试

```bash
# 设置详细日志
npx tsx cli/src/main.ts config set general.log_level debug

# 查看日志
tail -f ~/.codeengine/logs/codeengine.log

# 同时终端显示日志
npx tsx cli/src/main.ts run 2>&1 | tee codeengine-debug.log
```

### 14.3 清理与恢复

```bash
# 清理编译缓存
npx tsc --build --clean

# 重置配置为默认
npx tsx cli/src/main.ts config reset

# 清理所有会话 (谨慎!)
rm -rf ~/.codeengine/data/*.db

# 清理日志
rm ~/.codeengine/logs/*.log

# 恢复最新快照
npx tsx cli/src/main.ts checkpoint restore $(
```

## 15. 常见问题

### Q1: 如何切换 LLM 模型？

修改 `~/.codeengine/config.toml` 中的 model 字段：

```toml
[providers.ollama]
model = "qwen3.6-35b"   # 改为需要的模型
```

```toml
[providers.openai]
id = "openai"
type = "openai_compatible"
name = "OpenAI"
model = "gpt-4o"
baseURL = "https://api.openai.com/v1"
apiKey = "sk-..."
maxTokens = 8192
```

### Q3: 如何提高编译速度？

1. 使用 `tsx` 直连运行，跳过编译步骤
2. 启用 `compaction` 减少上下文大小
3. 设置 `max_concurrent_requests = 1` 避免竞争
4. 确保编译工具路径正确

### Q4: 如何备份数据？

```bash
# 备份整个数据目录
tar -czf codeengine-backup-$(date +%Y%m%d).tar.gz ~/.codeengine/

# 备份数据库
cp ~/.codeengine/data/codeengine.db ~/.codeengine/data/codeengine.db.backup
```

### Q5: 如何查看项目支持的命令？

```bash
# 查看所有命令
npx tsx cli/src/main.ts help
```

编辑 `~/.codeengine/config.toml`：

```toml
[features]
```

### Q7: 如何查看磁盘使用情况？
